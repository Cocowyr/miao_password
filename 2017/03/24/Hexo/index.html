Hexo 是什么？

Hexo是一个快速，简单和强大的博客框架。 您在Markdown（或其他语言）中写文章，Hexo会在几秒钟内生成具有美丽主题的静态文件。

环境要求

安装Hexo很容易。 但是，您需要首先安装几个其他的东西：

  • Node.js
  • Git

    安装

    只需要一个命令来安装Hexo。 如果您遇到问题，找不到解决方案，请联系我，我会尽力解决。

设置

安装Hexo后，运行以下命令在目标<文件夹>中初始化Hexo。

一旦初始化，您的项目文件夹将如下所示：

_config.yml

站点配置文件。 您可以在此配置大多数设置。

package.json

应用数据。 EJS，Stylus和Markdown渲染器默认安装。 如果需要，您可以稍后卸载它们。

scaffolds

脚手架文件夹 当您创建一个新文章时，Hexo将新文件放在脚手架上。

source

源文件夹。 这是您将网站内容存放的位置。 Hexo忽略隐藏的文件和名称前缀为_（下划线）的文件或文件夹 - 除了_posts文件夹。 可渲染文件（例如Markdown，HTML）将被处理并放入（public folder）公用文件夹，而其他文件将被简单复制。

themes

主题文件夹。 Hexo通过将网站内容与主题相结合来生成一个静态网站。

使用备用配置

hero 命令的 –flag标签 可以指定一个自定义的配置文件路径，可以是YAML文件也可以是JSON文件，或者是多个用逗号隔开的YAML文件和JSON文件。

使用多个文件组合所有配置文件，并将合并的设置保存到_multiconfig.yml。 稍后的值优先。 它适用于任意数量的JSON和YAML文件，具有任意深度的对象。 请注意，列表中不允许有空格。

例如，在上面的例子中，如果foo：bar在custom.yml中，但是“foo”：“dinosaur”在custom2.json中，_multiconfig.yml将包含foo：dinosaur。

命令

init

初始化网站 如果没有提供文件夹，Hexo将在当前目录中设置网站。

new

创建一个新的文章。 如果没有提供布局，Hexo将使用_config.yml中的default_layout。 如果标题包含空格，请用引号括起来。

generate

生成静态文件。

publish

发布草稿

server

启动本地服务器。 默认情况下，这是http：// localhost：4000 /。

deploy

部署您的网站

render

渲染文件

migrate

从其他博客系统迁移内容。

clean

清除缓存文件（db.json）和生成的文件（public）。

list

列出所有路线。

version

可选命令

Safe mode

禁用加载插件和脚本。 如果在安装新插件后遇到问题，请尝试这样做。

Debug mode

将详细消息记录到终端和debug.log。 如果您遇到Hexo的任何问题，请尝试此操作。 如果您看到错误，请联系我。

Silent mode

输出端口静音。

Customize config file path

Uses a custom config file (instead of _config.yml). Also accepts a comma-separated list (no spaces) of JSON or YAML config files that will combine the files into a single _multiconfig.yml.

Display drafts

显示草稿文章（存储在source/_drafts文件夹中）。

Customize CWD

自定义当前工作目录的路径。
